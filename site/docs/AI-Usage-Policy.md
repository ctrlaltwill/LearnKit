# AI Usage Policy

Last modified: 07/03/2026

## Purpose

This policy explains how LearnKit's AI features should be used and what users can expect when generating study content.

## Scope

This policy applies to AI-assisted features in LearnKit, including:

- Card suggestion and generation from note content.
- AI chat assistance for study and review workflows.

## What AI Features Do

LearnKit AI features are designed to suggest draft study content, not final truth.

Depending on your settings and prompt, AI output may include:

- Basic, cloze, and other card suggestions.
- Question and answer drafts.
- Difficulty estimates and rationale text.
- Explanations for note content in a study context.

## Human Review Is Required

AI output can be wrong, incomplete, biased, or outdated.

Before saving or studying AI-generated cards, users should:

- Verify factual accuracy.
- Correct unclear wording.
- Remove unsafe or irrelevant suggestions.
- Ensure card difficulty and scope are appropriate.

LearnKit treats AI output as a draft for user review.

## Data Handling and Privacy

When AI features are used, LearnKit may send selected content to the configured AI provider in order to generate a response.

Depending on user settings, this can include:

- Note content.
- Note path or contextual metadata.
- Referenced image information when image support is enabled.
- User prompts and custom instructions.

Users are responsible for reviewing provider terms and privacy policies before enabling AI features.

API keys configured for AI providers are stored locally in the plugin folder at `.obsidian/plugins/sprout/configuration/api-keys.json`.

The folder name remains `sprout` for compatibility, even though the plugin name is LearnKit.

If you track `.obsidian` plugin files in Git, add only this file to `.gitignore` to avoid accidentally sharing credentials while letting other LearnKit configuration files sync across devices.

## Sensitive and Confidential Data

Do not submit content that is highly sensitive, regulated, or confidential unless you fully understand and accept the data handling implications.

Examples include:

- Personal health information.
- Financial account details.
- Passwords, API keys, or private credentials.
- Private client, workplace, or exam-restricted material.

## Copyright and Content Rights

Users must only submit content they have the right to use.

Users are responsible for ensuring AI-generated cards do not infringe copyright, violate licensing terms, or misuse protected materials.

## Safety and Educational Use

LearnKit AI features are intended for learning support only.

They are not a substitute for professional advice in medical, legal, financial, or safety-critical domains.

## Availability and Limits

AI features may change over time and can be affected by:

- Provider uptime and model changes.
- Rate limits or token limits.
- Cost controls set by the user or provider.

LearnKit does not guarantee uninterrupted AI availability or output quality.

## Policy Updates

This policy may be updated as AI features evolve.

Material updates will be reflected by changing the Last modified date on this page.
