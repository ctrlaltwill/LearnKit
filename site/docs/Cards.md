# Cards

Last modified: 17/03/2026

## Purpose

Cards are the core study unit in LearnKit. You can create them in your notes, format them for clear recall, and manage them across browser and review flows.

## Card types

- [Basic & Reversed Cards](./Basic-&-Reversed-Cards): direct question/answer recall
- [Cloze Cards](./Cloze-Cards): fill-in-the-blank recall in context
- [Image Occlusion](./Image-Occlusion): hide parts of images
- [Multiple Choice Questions](./Multiple-Choice-Questions): choose the correct option
- [Ordered Questions](./Ordered-Questions): remember sequences

## Build Flow

Two reliable approaches:

1. Write card syntax directly in notes.
2. Use Add flashcard actions from the context menu or command palette.

See [Creating Cards](./Creating-Cards), [Reading View](./Reading-View), [Reading View Styles](./Reading-View-Styles), and [Custom Reading Styles](./Custom-Reading-Styles).

## Edit Flow

You can edit cards:

- in the source note
- in [Card Browser](./Card-Browser)
- during review (edit action)

See [Editing Cards](./Editing-Cards).

## Organize Cards

- Use groups to organize cards.
- Use [Card Formatting](./Card-Formatting) for markdown, math, links, and images.
- Use [Flags](./Flags) for inline circle-flag tokens.

## Key Notes

- Broken or incomplete card syntax can cause cards to be skipped or parsed incorrectly.
- Card scheduling data is separate from note text (see [Backups](./Backups)).
